PLUGIN_VIRTUALLIBRARYCREATOR_SETTINGS_DAILYVLREFRESHTIME
	EN	Daily refresh time of virtual libraries

PLUGIN_VIRTUALLIBRARYCREATOR_SETTINGS_DAILYVLREFRESHTIME_DESC
	EN	LMS normally updates the list of matching virtual library tracks only after a <b>rescan</b> or a <b>server restart</b>.<br><br>However, if you rate, play or skip tracks between rescans/reboots, virtual libraries with parameters like <i>rating</i>, <i>playCount</i>, <i>date lastPlayed</i>, <i>skipCount</i>, <i>date lastSkipped</i> or <i>dynamic played skipped value</i> may not contain all matching tracks.<br><br>In these cases, you can use the <b>daily refresh</b> option. Once a day at the time that you set here, VLC will refresh/rebuild all enabled VLC virtual libraries with this option enabled, i.e. update the list of matching tracks. If your server is in sleep mode or switched off at this time, the refresh will happen as soon as the server wakes up or is switched on again.<br><br><b>Please note</b>: The larger your virtual libraries, the longer a refresh takes. It is therefore better to limit the <i>Daily Refresh</i> option to a few select virtual libraries that include one or more of the above parameters and to set a time when the server load is normally low and you are not using LMS.

PLUGIN_VIRTUALLIBRARYCREATOR_SETTINGS_DISPLAYVLIDS
	EN	Display virtual library IDs

PLUGIN_VIRTUALLIBRARYCREATOR_SETTINGS_DISPLAYVLIDS_DESC
	EN	The <b>Virtual Library ID</b> is the (permanent) ID used to register a virtual library in LMS. It starts with “PLUGIN_VLC_VLID_”. The rest is derived from the filename of your virtual library when you <b>first</b> saved it. Changing your virtual library <b>name</b> does <b>not</b> change the <b>ID</b>. VLC handles all filenames and virtual library IDs.<br>Most users will never have a use for the <b>Virtual Library ID</b>. So, by default, it's not displayed on the <i>Edit</i> page. If you really need it displayed (e.g. for a manually customized dynamic playlist), you can enabled it here.

	EN	Refresh plugin caches

PLUGIN_VIRTUALLIBRARYCREATOR_SETTINGS_REFRESHCACHES_DESC
	EN	The <i>Virtual Libraries Creator</i> plugin caches lists of artists, genres, composers, audio file types and release types. They are automatically refreshed after an LMS rescan or a plugin update.<br><br>If you think there's a problem with the VLC plugin caches (e.g. missing artists or genres in the template), this button allows you to force a refresh of all plugin caches.
